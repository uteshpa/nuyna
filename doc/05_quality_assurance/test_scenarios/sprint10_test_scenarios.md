# nuyna - Sprint 10 テストシナリオ

**プロジェクト:** nuyna (Creator's Privacy Toolkit)  
**スプリント:** Sprint 10 - 機能検証とUI改善  
**作成日:** 2026年1月8日  
**作成者:** Manus AI

---

## 1. はじめに

本ドキュメントは、Sprint 10で実施するテストシナリオを定義します。Sprint 10の主な目的は、Sprint 9で実装されたパフォーマンス最適化の効果を検証し、アプリケーションのコア機能が仕様通りに動作することを確認することです。また、プロダクトの新しい方向性である「シンプルさ」と「速さ」を体現するためのUI改善の検証も行います。

---

## 2. テスト対象デバイス

テストは、性能の異なる複数の実機で行うことを推奨します。

- **ハイエンド:** iPhone 15, Google Pixel 8 Pro など
- **ミドルレンジ:** iPhone SE (第3世代), Google Pixel 7a など
- **ローエンド:** iPhone 11, 5年以上前のアンドロイド端末など

---

## 3. テストシナリオ

### 3.1. パフォーマンスベンチマーク (DoD-9-04 検証)

**目的:** 1分間の動画を30秒以内に処理できるか、という最重要要件の達成度を測定する。

| テストケースID | テスト内容 | 期待される結果 | 優先度 |
| :--- | :--- | :--- | :--- |
| **TC-PERF-01** | **基準性能測定（1分動画）**<br>各テストデバイスで、1分間のサンプル動画（1080p, 30fps）を処理し、開始から完了までの時間を正確に測定する。 | ハイエンド機で30秒以内、ミドルレンジ機で45秒以内に処理が完了する。 | **Critical** |
| **TC-PERF-02** | **メモリ使用量プロファイリング**<br>TC-PERF-01の実行中、Flutter DevToolsを用いてメモリ使用量を監視する。 | アプリがクラッシュせず、メモリ使用量が異常に増加しないこと。 | **High** |
| **TC-PERF-03** | **CPU使用率プロファイリング**<br>TC-PERF-01の実行中、Flutter DevToolsを用いてCPU使用率を監視し、ボトルネックを特定する。 | 特定の処理（顔検出、エンコード等）にCPUが集中していることを確認し、レポートと一致するか検証する。 | **High** |
| **TC-PERF-04** | **ストレステスト（連続処理）**<br>3本の異なる動画を連続で処理させ、アプリの安定性と一貫したパフォーマンスを確認する。 | アプリがクラッシュせず、処理時間が回を追うごとに極端に劣化しないこと。 | **Medium** |

### 3.2. 顔検出とぼかし機能の検証

**目的:** 様々な条件下で、顔検出とぼかし機能が正確かつ確実に動作することを確認する。

| テストケースID | テスト内容 | 期待される結果 | 優先度 |
| :--- | :--- | :--- | :--- |
| **TC-FACE-01** | **単一の顔（正・横顔）**<br>1人の顔がはっきりと映っている動画を処理する。 | 顔全体が正確に追跡され、フレームアウトするまでぼかしが適用され続けること。 | **High** |
| **TC-FACE-02** | **複数の顔**<br>5人以上の顔が同時に映っている動画を処理する。 | すべての顔が検出され、ぼかしが適用されること。 | **High** |
| **TC-FACE-03** | **悪条件下での検出**<br>低照度、逆光、手ブレの激しい動画を処理する。 | 検出精度が低下することは許容するが、検出できた顔にはぼかしが適用されること。 | **Medium** |
| **TC-FACE-04** | **部分的に隠れた顔**<br>マスク、サングラス、帽子などで顔の一部が隠れている動画を処理する。 | 顔として認識できる範囲で、検出・ぼかしが試みられること。 | **Medium** |
| **TC-FACE-05** | **高速な動き**<br>被写体が素早く動く、またはカメラが激しくパンする動画を処理する。 | 処理落ちせず、追従できる範囲でぼかしが適用されること。 | **Medium** |
| **TC-FACE-06** | **偽陽性（False Positive）テスト**<br>顔のように見える模様、絵画、ポスターなどが含まれる動画を処理する。 | 人間の顔以外を誤って検出しないこと。 | **Low** |
| **TC-FACE-07** | **顔が存在しない動画**<br>風景など、顔が一切含まれない動画を処理する。 | 顔検出処理がスキップされ、メタデータ削除のみが高速に完了すること。 | **High** |

### 3.3. メタデータ削除機能の検証

**目的:** 動画ファイルから意図したメタデータが完全に削除されていることを確認する。

| テストケースID | テスト内容 | 期待される結果 | 優先度 |
| :--- | :--- | :--- | :--- |
| **TC-META-01** | **メタデータ削除の検証**<br>GPS位置情報、撮影日時、デバイス情報（機種名など）を含む動画を処理する。 | 処理後の動画をメタデータビューアで開き、上記の情報がすべて削除されていることを確認する。 | **High** |
| **TC-META-02** | **ファイルサイズの確認**<br>処理前後のファイルサイズを比較する。 | メタデータが削除された分、ファイルサイズが僅かに減少すること。 | **Low** |

### 3.4. プラットフォーム互換性テスト

**目的:** iOSとAndroidで、同等の機能とユーザー体験が提供されていることを確認する。

| テストケースID | テスト内容 | 期待される結果 | 優先度 |
| :--- | :--- | :--- | :--- |
| **TC-PLAT-01** | **UIレイアウトの一貫性**<br>異なる画面サイズのiOS/Androidデバイスで、UIのレイアウトが崩れていないことを確認する。 | すべてのデバイスで、ボタンやテキストが適切に表示されること。 | **High** |
| **TC-PLAT-02** | **ファイル選択・保存**<br>各OSの標準的な方法で動画ファイルを選択し、処理後の動画をギャラリーに保存できることを確認する。 | ファイルの読み込み、書き込みが正常に完了し、パーミッション要求も適切であること。 | **High** |

### 3.5. ユーザビリティ・UIテスト

**目的:** 「ワンタップ処理」を核とした、シンプルで直感的な操作性を検証する。

| テストケースID | テスト内容 | 期待される結果 | 優先度 |
| :--- | :--- | :--- | :--- |
| **TC-UI-01** | **ワンタップ処理フロー**<br>動画を選択した後、1回のタップで処理が開始されることを確認する。 | 冗長な確認画面や設定項目がなく、即座に処理が開始されること。 | **Critical** |
| **TC-UI-02** | **進捗表示の明確さ**<br>処理中に表示されるプログレスバーやインジケーターが、ユーザーにとって分かりやすいか確認する。 | 処理の進行状況が直感的に理解できること。 | **High** |
| **TC-UI-03** | **完了と共有**<br>処理完了後、保存または共有の操作が簡単に行えることを確認する。 | 数タップ以内で、目的のアクション（保存、SNS投稿など）が完了できること。 | **High** |
